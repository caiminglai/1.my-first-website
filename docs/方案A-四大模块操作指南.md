# 同物异名 · Obsidian 内容协作方案

> **目标**: Obsidian 作为内容生产后台，网站数据库作为展示前台。
> **支持模块**: 词条、概念对比
> **说明**: 职业解构/岗位拆解/概念抗体 因涉及多表关联，建议通过管理后台编辑

---

## 一、工作流程

```
Obsidian 编辑 Markdown 文件
         ↓
    Git 推送到服务器
         ↓
  Obsidian同步.js 自动触发
         ↓
  Obsidian同步到数据库.js 解析入库
         ↓
   网站 API 实时展示
```

---

## 二、Obsidian 仓库结构

```
同物异名/
├── 模板/              ← 存放 Markdown 模板
├── 词条/              ← 词条模块（会同步到数据库）
│   ├── 经济学/
│   ├── 医学/
│   ├── 法学/
│   ├── 计算机/
│   └── 其他学科/
└── 概念对比/          ← 概念对比模块（会同步到数据库）
```

> 注意：`概念抗体/`、`职业解构/`、`岗位拆解/` 这三个模块目前不支持 Obsidian 写入，如有需要请通过管理后台 http://localhost:3000/admin 编辑。

## 三、模板使用说明

### 模板1：词条（核心模块）

**文件存放位置**: `词条/学科名/词条名称.md`

**frontmatter 字段说明**（必须严格按此格式）：

```yaml
---
类型: 词条
词条ID: econ-001          # 唯一ID，不填则自动生成
学科: 经济学               # 必须是已有学科
热度: 1                    # 可选，默认0
创建时间: 2026-06-24      # 可选，自动生成
别名: ["边际效应", "Marginal Utility"]  # 跨学科别名，JSON数组
---
```

**正文格式**（使用以下标题标记）：

```markdown
# 词条名称（一级标题 = 名称）

**专业说法：** 每增加一单位消费所带来的额外满足程度。

**人话翻译：**
就像你吃包子。饿的时候第一个包子香得很，第二个还行...

**本质：** 增量满足递减规律

**提示：** 别被"边际"两个字唬住

**跨学科别名：**   ← 下面写 [[双链]] 格式
- [[饱和效应]]
- [[机会成本]]

**相关术语：**      ← 下面写 [[双链]] 格式
- [[沉没成本]]
```

**重要规则**:
- `学科` 必须是数据库里已存在的学科（数学、物理学、经济学、医学、心理学、计算机科学等）
- 文件名即为词条名称，`名称` 字段可省略
- `[[词条名称]]` 双链会自动写入图谱关系表

---

### 模板2：概念对比

**文件存放位置**: `概念对比/对比标题.md`

**frontmatter 字段说明**：

```yaml
---
类型: 概念对比
对比ID: cmp-001          # 唯一ID，不填则自动生成
关系类型: 关联性          # 可选：同义词/反义词/包含关系/相近概念/关联性
创建时间: 2026-06-24      # 可选，自动生成
---
```

**正文格式**：

```markdown
# 概念A名称 vs 概念B名称

## 概念A：边际效用

**所属学科：** 经济学

**人话翻译：** 再多吃一个包子的爽感

**表现症状：** 越买越不划算

**生活类比：** 吃包子

**解决办法：** 适可而止

## 概念B：沉没成本

**所属学科：** 经济学

**人话翻译：** 已经花出去的钱

**表现症状：** 舍不得放弃

**生活类比：** 电影票买了不好看也看完

**解决办法：** 及时止损

## 总结

**一句话区分：** 边际效用是未来还能爽多少，沉没成本是过去已经花掉的钱。
```

**重要规则**:
- 标题必须包含 `vs`（大小写均可），用于自动拆分两个概念
- `所属学科` 必须和数据库里的学科名称一致

## 四、Git 同步（本地 → 服务器）

### 4.1 首次初始化

```bash
# 在 Obsidian 仓库目录下执行
cd D:\同物异名
git init
git add .
git commit -m "初始提交"
git remote add origin root@你的服务器IP:/root/git-twym.git
git push -u origin main
```

### 4.2 日常使用

```bash
# 编辑完内容后
git add .
git commit -m "新增词条：通货膨胀"
git push origin main
```

### 4.3 服务器自动同步

服务器上已配置 `Obsidian同步.js`，Git push 后会自动：
1. 拉取最新 Obsidian 文件
2. 备份当前数据库
3. 解析 Markdown 入库
4. 重启后端服务

| `scripts/端口清理.bat` | 清理被占用的端口 |

## 六、字段对照表（数据库 vs Obsidian frontmatter）

### 词条

| 数据库字段 | Obsidian frontmatter | 说明 |
|-----------|---------------------|------|
| 词条ID | `词条ID` | 唯一标识，不填自动生成 |
| 学科 | `学科` | 必须与数据库已有学科一致 |
| 名称 | 文件名 / `# 标题` | 文件名即词条名 |
| 翻译 | `**人话翻译：**` | 正文小标题 |
| 本质 | `**本质：**` | 正文小标题 |
| 提示 | `**提示：**` | 正文小标题 |
| 跨学科别名 | `别名` | JSON数组格式 |
| 热度 | `热度` | 可选 |

### 概念对比

| 数据库字段 | Obsidian frontmatter | 说明 |
|-----------|---------------------|------|
| 对比ID | `对比ID` | 唯一标识 |
| 标题 | `# 标题` | 必须含 `vs` |
| 关系类型 | `关系类型` | 可选 |
| A概念名称 | `## 概念A：...` | 含 `概念A：` |
| A概念学科 | `**所属学科：**`（A后） | 正文小标题 |
| A概念平述 | `**人话翻译：**`（A后） | 正文小标题 |
| A概念症状 | `**表现症状：**`（A后） | 正文小标题 |
| A概念类比 | `**生活类比：**`（A后） | 正文小标题 |
| A概念修正 | `**解决办法：**`（A后） | 正文小标题 |
| B概念... | 同上结构 | B概念段落 |
| 总结 | `**一句话区分：**` | 正文小标题 |
