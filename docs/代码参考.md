# 同物异名（科学不装）-- 代码参考

> 更新日期：2026-07-04
> 技术栈：React 19 + Vite 7 (前端) | Express.js + SQLite + Meilisearch (后端)

---

## 1. 整体架构

```
用户浏览器
    │
    ▼
React 前端 (Port 3001)
    │  HTTP/REST
    ▼
Express 后端 (Port 3000)
    │  SQL            │  HTTP
    ▼                  ▼
SQLite 数据库     Meilisearch (Port 7700)
(同物异名.db)     全文搜索引擎
```

请求链路：`中间件层(安全/限流) → 路由层(分发) → 服务层(业务逻辑) → 数据库层(SQL操作)`

API 路径双重挂载：同时支持 `/api/v1/` 和 `/twym/api/v1/`。

### 2.2 API 客户端架构

前端采用两层 API 抽象：

- **client.ts**：底层 HTTP 封装，自动重试 3 次、30 秒超时、统一错误处理、自动解包 `{ success, data }` 响应
- **services.ts**：上层业务函数，每个后端接口对应一个具名函数，组件只调用这些函数

调用链路：`组件 → services.ts 函数 → client.ts apiClient → fetch → 后端API`

### 2.3 数据 Hooks

| Hook | 功能 | 缓存策略 |

| `useTerms()` | 词条列表 + 按学科分组 | 模块级共享缓存，TTL 2 分钟 |

| `useFavorites()` | 用户收藏管理 | localStorage |
| `useTheme()` | 主题切换 (亮色/暗色) | - |

### 3.2 中间件层

| 中间件 | 功能 | 限制 |
|--------|------|------|
| `rateLimiter` | 通用限流 | 1000 请求/分钟 |
| `searchRateLimiter` | 搜索限流 | 600 请求/分钟 |
| `adminRateLimiter` | 管理接口限流 | 300 请求/分钟 |
| `securityHeaders` | 安全响应头 | X-Content-Type-Options, X-Frame-Options 等 |
| `bodySizeLimit` | 请求体大小限制 | 1MB |
| `adminAuth` | 管理员 Token 认证 | x-admin-token 请求头 |
| `sqlInjectionCheck` | SQL 注入检测 | - |

注意：限流器基于内存 Map，仅单进程模式有效。PM2 cluster 模式需改用 Redis 共享存储。

### 3.4 缓存设计

缓存服务 (services/缓存.js) 核心 API：

- `cache.get(key)` / `cache.set(key, value, ttlMs)` — 读写缓存
- `cache.invalidate(...keys)` — 清除指定缓存
- `cache.invalidatePrefix(prefix)` — 按前缀批量清理
- `cache.remember(key, fetcher)` — 缓存或回源

默认 TTL：词条/图谱 5 分钟，统计数据 30 秒，用户提交 1 分钟。

### 3.5 搜索降级机制

Meilisearch 服务 (services/Meilisearch服务.js)：
- 搜索优先走 Meilisearch，不可用时自动降级为 SQLite LIKE
- 增删改词条时自动同步到 Meilisearch 索引
- 全量同步：`npm run sync:meilisearch`

## 4. 数据库设计

### 4.1 基本信息

- 路径：`data/同物异名.db`（可通过环境变量 `DB_PATH` 覆盖）
- 驱动：better-sqlite3（同步 API，直接读写磁盘文件）
- WAL 模式：已启用，支持读写并发
- 外键约束：已启用
- 中文表名列名，启动时自动检测并添加缺失列

### 4.2 核心表结构

**词条表 (词条)**

| 字段 | 类型 | 说明 |
|------|------|------|
| 词条ID | TEXT | 主键 |
| 学科 | TEXT | 所属学科名称（中文） |
| 名称 | TEXT | 词条名称 |
| 翻译 | TEXT | 通俗翻译 |
| 本质 | TEXT | 本质含义 |
| 提示 | TEXT | 记忆小贴士 |
| 跨学科别名 | TEXT | JSON 数组 |
| 热度 | INTEGER | 是否热门 |
| 创建时间 | DATETIME | 创建时间 |

**学科表 (学科)**

| 字段 | 类型 | 说明 |
|------|------|------|
| 学科ID | TEXT | 主键（与学科名称一致） |
| 学科名称 | TEXT | 学科中文名 |
| 学科颜色 | TEXT | 十六进制颜色码 |
| 学科描述 | TEXT | 学科说明 |
| 显示顺序 | INTEGER | 排序序号 |

**图谱关系表 (图谱关系)**

| 字段 | 类型 | 说明 |
|------|------|------|
| 关系ID | INTEGER | 自增主键 |
| 源节点ID | TEXT | 源词条 ID |
| 目标节点ID | TEXT | 目标词条 ID |
| 关系标签 | TEXT | 关系描述（如"同物异名"） |

**概念对比表 (概念对比)**：存储两个概念的对比信息，包含各自的通俗解释、表现、类比、解决方法。

**情景还原表 (情景还原 + 情景对话)**：存储真实场景和对话，揭示术语包装的真相。

**高薪技术岗表**：`岗位分类 → 岗位 → 岗位学习阶段 → 岗位技能 / 岗位学习资源 / 岗位项目 / 岗位知识卡片`

### 4.3 表关系

```
学科 ──1:N──→ 词条 ──N:1──→ 图谱关系
                 │
                 │ N:1
                 ▼
             概念对比 ──1:N──→ 情景还原
```

## 5. 关键模块说明

### 5.1 数据库层 (db/入口.js)

导出接口：`initDB`, `prepare`(预编译 SQL), `exec`(执行 SQL), `reloadDB`(重新加载), `markDirty`(标记写入并清缓存), `closeDb`。

设计原则：直接读写原始数据库文件，写入操作立即同步到磁盘，无定时保存机制，避免覆盖外部修改。

### 5.2 缓存 Key 规范 (utils/缓存键.js)

命名规则：`模块_功能_参数`。示例：`terms_list_1_50_math`（学科 math 的词条列表第 1 页）、`terms_hot`（热门词条）、`graph_nodes`（图谱节点）。

## 7. 错误码
