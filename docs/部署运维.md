## 4. 部署命令

### 快速上线流程

```bash
# 1. 进入项目
cd /var/www/my-first-website

# 2. 重装依赖（有新增依赖时）
cd backend && rm -rf node_modules && npm install && cd ..

# 3. 重启后端（改 .env 后必须加 --update-env）
pm2 restart my-website --update-env

# 4. 检查状态
pm2 list
pm2 logs my-website --lines 20

# 5. 数据库若清空，重新初始化
cd backend && node init-db.js && pm2 restart my-website

# 6. 全量同步数据到 Neo4j
cd backend && npm run sync:neo4j
```

### PM2 常用命令

```bash
pm2 list                              # 查看进程状态
pm2 restart my-website --update-env   # 重启并更新环境变量
pm2 logs my-website --lines 50        # 查看日志
pm2 save && pm2 startup               # 设置开机自启
pm2 stop my-website                    # 停止
```

注意：不要使用 `pm2 start -i max`（cluster 模式），限流器基于内存 Map，cluster 模式下会失效。

### Nginx 命令

```bash
nginx -t                              # 检查配置语法
systemctl reload nginx                # 热重载
systemctl restart nginx               # 重启
tail -f /var/log/nginx/error.log      # 查看错误日志
```

```bash
cd app && npm run build
```

**Neo4j 备份：**


## 5. Nginx 配置 (kxbz365.cn)

配置文件路径：`/etc/nginx/conf.d/my-website.conf`

```nginx
server {
    listen 443 ssl;
    server_name kxbz365.cn www.kxbz365.cn;

    ssl_certificate /etc/nginx/ssl/kxbz365.cn.pem;
    ssl_certificate_key /etc/nginx/ssl/kxbz365.cn.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
    ssl_prefer_server_ciphers on;

    # 根路径重定向到 /twym/
    location = / {
        return 302 /twym/;
    }

    # /twym 补斜杠
    location = /twym {
        return 301 /twym/;
    }

    # 前端静态文件
    location /twym/ {
        alias /var/www/my-first-website/dist/;
        index index.html;
        try_files $uri $uri/ @twym_fallback;
    }

    location @twym_fallback {
        root /var/www/my-first-website/dist;
        try_files /index.html =404;
    }

    # 后端 API 代理 (/twym/api/ → 后端 /api/)
    location /twym/api/ {
        rewrite ^/twym/api/(.*)$ /api/$1 break;
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 管理后台静态 HTML
    location ~ ^/twym/admin/.*\.html$ {
        root /var/www/my-first-website/backend/public;
        rewrite ^/twym/admin/(.*)$ /$1 break;
        try_files $uri =404;
    }

    # 管理后台代理
    location /twym/admin {
        rewrite ^/twym/admin/?$ /admin break;
        rewrite ^/twym/admin/(.*)$ /admin/$1 break;
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 直接 API 访问
    location /api/v1/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name kxbz365.cn www.kxbz365.cn;
    return 301 https://$server_name$request_uri;
}
```

## 7. 问题排查

### 7.1 本地开发问题

| 症状 | 根因 | 解决 |
|------|------|------|
| npm run dev 报错 ENOENT | 没在 app/ 文件夹执行 | `cd E:\website\1.my-first-website\app` |
| npm start 报错 ENOENT | 没在 backend/ 文件夹执行 | `cd E:\website\1.my-first-website\backend` |
| 后端 500 / 请求超时 | 后端卡住或数据库异常 | Ctrl+C 后重新 npm start |
| 页面能开但无数据 | API 超时 | Ctrl+F5 + 重启后端 |
| better-sqlite3 编译失败 | 缺 C++ 编译器或 Node 版本不对 | 装 VS Build Tools 或确认 Node v20 |
| 端口占用 EADDRINUSE | 旧进程没释放 | `taskkill /f /im node.exe` |

### 7.2 服务器问题

| 症状 | 根因 | 解决 |
|------|------|------|
| 网页打不开 | Nginx 没启动 | `systemctl start nginx && nginx -t` |
| 页面能开但没数据 | 后端挂了 | `pm2 list` → `pm2 restart my-website` |
| 登录失败 | .env 改了没重启 | `pm2 restart my-website --update-env` |
| 页面图片/样式丢失 | 路径没加斜杠 | 检查网址 `/twym/` |
| 改完代码没变化 | 没重启或缓存 | Ctrl+F5 + pm2 restart |
| 502 Bad Gateway | 后端端口不匹配 | `pm2 list` + `ss -tlnp \| grep 3000` |
| 图谱页面无数据 | Neo4j 未启动或连接失败 | `systemctl status neo4j` 或 `docker ps \| grep neo4j`，检查 .env 中 NEO4J_URI 配置 |
| Neo4j 数据不同步 | 同步任务未执行 | 手动执行 `npm run sync:neo4j`，检查后端日志 |

### 7.3 备案问题

现象："域名暂时无法访问"（阿里云备案拦截）。

原因：备案接入未完成，管局要求暂停解析。

解决步骤：
1. 登录阿里云 DNS 控制台，暂停 kxbz365.cn 和 www.kxbz365.cn 的解析记录
2. 回到阿里云备案平台，继续填写网站接入信息并提交审核
3. 等待审核：阿里云初审 1 个工作日 → 短信核验 → 管局审核 3-5 个工作日
4. 审核通过后恢复 DNS 解析

## 8. 调试铁律

1. 本地启动必须先 cd 进对应文件夹（app/ 或 backend/）
2. 改 .env 后必须 `pm2 restart --update-env`
3. Node 版本必须 v20.20.2，不能回退到 v22
4. Windows 本地重装 node_modules 后如果 better-sqlite3 报错，确认已装 VS C++ Build Tools
5. 服务器只执行三件事：接收代码、安装依赖、重启服务。不在服务器上临时改代码
6. 修改后端路由代码后必须重启服务（先 netstat 找端口 PID → taskkill 杀旧进程 → 启动新进程）
7. 所有服务器操作前先备份配置文件：`cp /etc/nginx/conf.d/xxx.conf /etc/nginx/conf.d/xxx.conf.bak`
