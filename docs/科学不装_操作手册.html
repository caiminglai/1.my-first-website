一、项目概述与架构

1.1 项目定位

科学不装是一个跨学科概念还原库，核心目标是将各学科、行业中的专业术语翻译成老百姓能听懂的大白话。

1.2 四大模块

模块内容数据库表
词条专业术语 → 人话翻译词条
概念抗体纠正常见概念误解概念抗体
职业解构拆解职业学习路径职业解构
岗位拆解行业黑话翻译 + 薪资真相岗位
概念对比两个概念并排对比概念对比

1.3 技术架构

Obsidian(写词条) --Git Push--> 服务器Git(裸仓库) --sync.sh--> SQLite(网站数据) --> 网站前端(展示)

1.4 服务器信息

四、Git 同步配置

4.1 本地初始化 Git

打开命令行（Win+R → cmd），执行：

cd E:\website\科学不装
git init
git add .
git commit -m "科学不装初始词条库"

4.2 服务器创建裸仓库

SSH 登录服务器（your_server_ip），执行：

mkdir -p /root/git-kxub.git
cd /root/git-kxub.git
git init --bare

4.3 本地关联远程

cd E:\website\科学不装
git remote add origin root@your_server_ip:/root/git-kxub.git
如果提示 remote origin already exists，执行：
git remote set-url origin root@your_server_ip:/root/git-kxub.git

4.4 首次推送

git push -u origin main

# 如果 main 分支不存在，试试：
git push -u origin master

4.5 Git 插件自动备份配置

设置 → 第三方插件 → Git → 选项：

设置项值
Auto backup interval (minutes)5
Auto commit message自动备份: {{date}}
Auto push开启
Auto save before commit开启
Pull updates on startup开启
配置完成后，Obsidian 每5分钟自动执行：git add → git commit → git push

五、服务器部署

5.1 上传解析脚本

将 parse-md-to-sqlite-v2.js 上传到服务器：

scp parse-md-to-sqlite-v2.js root@your_server_ip:/var/www/my-first-website/backend/

5.2 上传同步脚本

将 sync-v2.sh 上传到 /root/，并赋予执行权限：

chmod +x /root/sync-v2.sh

5.3 安装依赖

cd /var/www/my-first-website/backend
npm install better-sqlite3

5.4 克隆仓库

cd /root
git clone /root/git-kxub.git kxub-obsidian

5.5 手动同步

/root/sync-v2.sh

脚本执行流程：

[1] 拉取 Obsidian 仓库 → [2] 备份数据库 → [3] 解析 Markdown → [4] 清理旧备份 → [5] 重启后端

5.6 自动同步（可选）

设置定时任务，每5分钟自动同步：

crontab -e

# 添加一行：
*/5 * * * * /root/sync-v2.sh >> /var/log/twym-sync.log 2>&1

5.7 Nginx 配置

server {
    listen 80;
    server_name your_server_ip;

    location /twym/ {
        root /var/www/my-first-website/dist/;
        try_files $uri /twym/index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /admin {
        proxy_pass http://127.0.0.1:3000;
    }
}

六、数据库导出/导入

6.1 SQLite → Obsidian Markdown（导出）

将数据库内容导出为 Markdown 文件到 Obsidian：

脚本位置：E:\website\1.my-first-website\data\db-to-obsidian.js

// 配置
const DB_PATH = 'E:\\website\\1.my-first-website\\data\\tongwuyiming.db';
const OBSIDIAN_PATH = 'E:\\website\\科学不装';

// 运行
cd E:\website\1.my-first-website\data
node db-to-obsidian.js

6.2 建立词条关联（双链）

导出后词条没有关联，需要运行关联脚本：

cd E:\website\1.my-first-website\data
node auto-link.js

关联规则：

  • 规则1：YAML 手动指定的 相关术语 转成 [[ ]]
  • 规则2：别名匹配（跨学科关联）
  • 规则3：同学科自动关联（每个词条关联最多3个同领域词条）

6.3 Obsidian Markdown → SQLite（导入）

Obsidian 写完后同步到服务器数据库：

// 服务器执行
/root/sync-v2.sh

支持五大模块自动识别：

八、故障排查

8.1 常见问题

症状原因解决
Git 插件报错：not a git repository 仓库未初始化 Git git init → git commit
推送失败：remote rejected 远程仓库未创建 服务器执行 git init --bare
同步后网站无更新 后端未重启 pm2 restart my-website
词条显示为0 数据库为空/路径错误 检查 data/tongwuyiming.db 是否存在
解析脚本报错 better-sqlite3 未安装 npm install better-sqlite3
图谱无连线 无 [[双链]] 运行 auto-link.js 或手动添加
插件安装失败 网络问题 换下载源或手动安装

8.2 常用命令速查

Git 操作
