## 项目愿景

建立"认知基础设施"——将复杂世界翻译成人能理解的结构。核心目标：知识平民化、通俗化，把专业术语翻译成日常用语，反对语言障碍与权威壁垒。

核心洞察：不是词条数量，而是"发现不同领域在重复同一种结构"。例如控制论的"负反馈"、经济学的"市场调节"、生物的"内稳态"，本质都属于"系统自稳定"。

### 四大模块

1. **科学不装（同物异名网站）**：把专业术语翻译成"说人话"，覆盖 10 大学科，已收录约 1,069 个词条
2. **概念抗体**：三步还原法（物理/数学/权利还原），识别五类忽悠，引导认知升级
3. **行业分析**：认知地图、看门狗经济学、鬼话拆解、检测工具
4. **岗位拆解**：已收录约 26 个高薪岗位 SOP，降低职业信息黑箱

## 快速启动

```bash

cd E:\website\1.my-first-website\backend
npm install
npm start
# → http://localhost:3000

# 终端 2：前端
cd E:\website\1.my-first-website\app

# 终端 3：Meilisearch（可选，搜索功能需要）
```

## 数据库概览

### SQLite 关系数据库

| 数据库 | 表数量 | 记录数 | 说明 |
|--------|--------|--------|------|
| 同物异名.db | 24 | 17,206 | 主数据库：词条、学科、图谱关系、概念对比、情景还原、岗位等 |
| 学科词库.db | 19 | 8,198 | 学科专业术语词库 |
| 语义词典.db | 3 | 8,598 | 语义关系词典 |

数据库设计原则：文件即数据库（直接读写磁盘文件），WAL 模式，中文表名列名，启动时自动迁移缺失列。

### Neo4j 图数据库（双库架构）

采用 **SQLite 精加工工作台 + Neo4j 知识图谱展示** 的双库架构：

- **SQLite 层**：负责数据精加工、业务逻辑处理、管理后台操作，作为唯一数据源
- **Neo4j 层**：专注知识图谱可视化展示、图查询、路径分析，数据由 SQLite 同步而来

**数据规模**：

| 指标 | 数量 | 说明 |
|------|------|------|

**学科子领域映射**：19 个学科词库表映射至 10 个核心学科

## 社区协作机制

采用三层协作机制，支持开放式内容共建：

| 层级 | 机制 | 说明 |
|------|------|------|
| 第一层 | 学科映射管理 API | 管理19个词库表到10个核心学科的映射关系，支持动态调整学科归类 |
| 第二层 | 词条纠错建议 API | 用户可提交词条纠错、补充建议，管理员审核后生效 |
| 第三层 | Obsidian 内容协作 | 基于 Obsidian 的双向链接笔记系统，支持 Markdown 格式的内容批量导入导出，实现知识库与个人笔记的双向同步 |

相关脚本：
- `npm run sync:to-obsidian` — 数据库同步到 Obsidian
- `npm run sync:from-obsidian` — Obsidian 同步到数据库
