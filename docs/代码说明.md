## 1. 项目概述

### 1.1 项目简介

**同物异名**是一个跨学科概念还原库，旨在揭示同一概念在不同学科中的不同命名，帮助用户理解看似复杂术语的本质。

### 1.2 项目特性

- **概念还原**: 将专业术语还原为通俗易懂的解释
- **跨学科关联**: 展示同一概念在不同学科中的别名
- **知识图谱**: 可视化概念间的关联关系
- **AI助手**: 基于RAG的AI问答功能
- **情景还原**: 真实场景中的术语应用示例

## 5. 数据库设计

### 5.1 数据库文件

- **路径**: `data/tongwuyiming.db`（相对路径，可通过环境变量 `DB_PATH` 覆盖）
- **类型**: SQLite 3
- **驱动**: better-sqlite3（同步 API，直接读写磁盘文件）
- **WAL 模式**: 已启用，支持读写并发
- **外键约束**: 已启用

### 5.1.1 设计原则

1. **文件即数据库**：`tongwuyiming.db` 是唯一数据源，所有读写直接操作此文件
2. **无内存缓存层**：不使用 sql.js 内存数据库 + 定时回写，避免外部修改被覆盖
3. **中文表名列名**：所有表名和列名均为中文，与业务概念一致
4. **自动迁移**：启动时自动检测并添加缺失的列，保证向前兼容

## 7. 关键模块说明

### 7.1 前端关键模块

#### 7.1.1 API客户端层 (client.ts + services.ts)

前端采用**两层API抽象**：底层 `client.ts` 封装HTTP通信，上层 `services.ts` 提供业务函数。所有组件统一通过 service 函数调用后端，不直接使用原生 fetch。

**client.ts — HTTP请求封装**:

```typescript
// 核心请求函数
async function request<T>(
  method: string,      // HTTP方法
  path: string,        // API路径
  body?: unknown,      // 请求体
  timeout?: number     // 超时时间(ms)
): Promise<T>

// 导出方法
export const apiClient = {
  get: <T>(path, timeout?) => request<T>('GET', path, undefined, timeout),
  post: <T>(path, body?, timeout?) => request<T>('POST', path, body, timeout),
  put: <T>(path, body) => request<T>('PUT', path, body),
  del: <T>(path) => request<T>('DELETE', path),
}
```

**特性**: 自动重试3次、30秒默认超时、统一错误处理、自动解包 `{ success, data }` 响应（直接返回 `data`）。

**services.ts — 业务API函数**:

```typescript
// 每个后端接口对应一个具名函数，组件只调用这些函数
export async function getTerms(page?, pageSize?): Promise<Term[]>
export async function searchTerms(q, limit?): Promise<SearchResult[]>
export async function getDisciplines(): Promise<Discipline[]>
export async function getJobCategories(): Promise<Category[]>
export async function getDetailedJobs(): Promise<JobItem[]>
// ... 更多业务函数
```

**调用链路**: `组件` → `services.ts 函数` → `client.ts apiClient` → `fetch` → `后端API`

```javascript
  initDB,       // 初始化数据库
  saveDB,       // 保存数据库到磁盘
  reloadDB,     // 重新加载数据库
  prepare,      // 预处理SQL语句
  exec,         // 执行SQL
  markDirty,    // 标记有写入，清除缓存
  DB_PATH,      // 数据库文件路径
  closeDb,      // 关闭连接
}
```

**设计原则**:
- 直接读写原始数据库文件
- 写入操作立即同步到磁盘
- 无定时保存机制，避免覆盖外部修改

#### 7.2.2 内存缓存 (services/cache.js)

```javascript
// 核心API
cache.get(key)                    // 读取缓存

cache.remember(key, fetcher)       // 缓存或回源
```
