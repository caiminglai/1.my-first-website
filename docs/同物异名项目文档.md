# 第一部分：项目愿景与定位

## 1.1 核心理念

建立"认知基础设施"——将复杂世界翻译成人能理解的结构。

> 核心目标：知识平民化、通俗化，把专业术语翻译成日常用语，反对语言障碍与权威壁垒。

核心学科是基础学科（数学、物理、化学、生物、金融、法律、医学等）——普通人听到术语会被吓住的领域。

## 1.2 核心模块

### 科学不装（同物异名网站）

把专业术语翻译成"说人话"。覆盖学科：数学、物理、化学、生物、计算机、经济学、法律政策、医学心理、控制论、跨学科概念。当前已收录约 1,069 个词条。

### 概念抗体

三步还原法：物理还原 → 数学还原 → 权利还原。

五类忽悠识别：修辞堆砌型、缩写恐吓型、新旧换瓶型、权威绑架型、数学恐吓型。

认知升级路径：术语消费者 → 类比翻译者 → 结构识别者 → 权利分析者 → 概念构建者。

### 行业分析

包含认知地图、看门狗经济学、造血指南、认知偏差、鬼话拆解、检测工具。

### 岗位拆解

当前已收录约 26 个高薪岗位 SOP，目标是降低职业信息黑箱。

## 1.3 核心洞察——结构映射

不是词条数量，而是"发现不同领域在重复同一种结构"。

例如：控制论的"负反馈"、经济学的"市场调节"、生物的"内稳态"、医学的"血糖调节"，本质都属于"系统自稳定"。

## 1.4 教育问题洞察

很多学习困难不是人笨，而是学习入口反人性——形式化过早。更符合认知规律的路径：直觉 → 类比 → 图像 → 操作 → 现象 → 模型 → 形式化 → 严格证明。

## 1.5 商业化建议

不要无限堆词条，重点做传播节点：鬼话拆解、AI 职业分析、AI 术语翻译、职业风险扫描。

内容漏斗：短内容传播 → 网站沉淀 → 建立信任 → AI 工具产品化。

### 快速上线流程
```bash
# 1. 进入项目
cd /var/www/my-first-website

# 2. 重装依赖（有新增依赖时）
cd backend && rm -rf node_modules && npm install && cd ..

pm2 restart my-website --update-env

# 4. 检查状态
pm2 list
```

## 4.5 Nginx 配置（your_domain.com 部分）

```nginx
server {
    listen 443 ssl;
    server_name your_domain.com www.your_domain.com;

    ssl_certificate /etc/nginx/ssl/your_domain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/your_domain.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
    ssl_prefer_server_ciphers on;

    location = / {
        return 302 /twym/;
    }

    location = /twym {
        return 301 /twym/;
    }

    location /twym/ {
        alias /var/www/my-first-website/dist/;
        index index.html;
        try_files $uri $uri/ @twym_fallback;
    }

    location @twym_fallback {
        root /var/www/my-first-website/dist;
        try_files /index.html =404;
    }

    location /twym/api/ {
        rewrite ^/twym/api/(.*)$ /api/$1 break;
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location ~ ^/twym/admin/.*\.html$ {
        root /var/www/my-first-website/backend/public;
        rewrite ^/twym/admin/(.*)$ /$1 break;
        try_files $uri =404;
    }

    location /twym/admin {
        rewrite ^/twym/admin/?$ /admin break;
        rewrite ^/twym/admin/(.*)$ /admin/$1 break;
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/v1/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

server {
    listen 80;
    server_name your_domain.com www.your_domain.com;
    return 301 https://$server_name$request_uri;
}
```

# 第五部分：问题排查

## 5.1 your_domain.com 备案问题

### 问题现象
"域名暂时无法访问"（阿里云备案拦截）

### 根因
备案接入未完成，管局要求暂停解析。

### 解决步骤
1. 登录阿里云 DNS 控制台，暂停 your_domain.com 和 www.your_domain.com 的解析记录
2. 回到阿里云备案平台，继续填写网站接入信息并提交审核
3. 等待审核：阿里云初审 1 个工作日 → 短信核验 → 管局审核 3-5 个工作日
4. 审核通过后恢复 DNS 解析

### 备案信息
| 项目 | 值 |
|------|-----|
| ICP 备案号 | your_icp_number |
| 备案主体 | your_name |
| 网站名称 | 科学不装 |
| 域名 | your_domain.com |
| 接入服务商 | 阿里云 |

## 5.2 本地开发问题

| 症状 | 根因 | 解决 |
|------|------|------|
| npm run dev 报错 ENOENT | 没在 app/ 文件夹执行 | `cd E:\website\1.my-first-website\app` |
| npm start 报错 ENOENT | 没在 backend/ 文件夹执行 | `cd E:\website\1.my-first-website\backend` |
| 后端 500 / 请求超时 | 后端卡住或数据库异常 | Ctrl+C → 重新 npm start |
| 页面能开但无数据 | API超时 | Ctrl+F5 + 重启后端 |
| better-sqlite3 编译失败 | 缺C++编译器或Node版本不对 | 装 VS Build Tools 或确认 Node v20 |
| 端口占用 EADDRINUSE | 旧进程没释放 | `taskkill /f /im node.exe` |

## 5.3 服务器问题

| 症状 | 根因 | 解决 |
|------|------|------|
| 网页打不开 | Nginx没启动 | `systemctl start nginx && nginx -t` |
| 页面能开但没数据 | 后端挂了 | `pm2 list` → `pm2 restart my-website` |
| 登录失败 | .env改了没重启 | `pm2 restart my-website --update-env` |
| 页面图片/样式丢失 | 路径没加斜杠 | 检查网址 `/twym/` |
| 改完代码没变化 | 没重启或缓存 | Ctrl+F5 + pm2 restart |

## 5.4 调试铁律

1. 本地启动必须先 cd 进对应文件夹（app/ 或 backend/）
2. 改 .env 后必须 pm2 restart --update-env
3. Node 版本必须 v20.20.2，不能回退到 v22
4. Windows 本地重装 node_modules 后如果 better-sqlite3 报错，确认已装 VS C++ Build Tools
5. 服务器只执行三件事：接收代码、安装依赖、重启服务。不在服务器上临时改代码
6. 修改后端路由代码后必须重启服务（先 netstat 找端口 PID → taskkill 杀旧进程 → 启动新进程）
7. 所有服务器操作前先备份配置文件：`cp /etc/nginx/conf.d/xxx.conf /etc/nginx/conf.d/xxx.conf.bak`
