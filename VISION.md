# 同物异名 · 愿景与路线图

> 建立"认知基础设施"——用图数据库将复杂世界的知识结构化为可探索的网络。

*注：项目中出现的"科学不装"为品牌名称占位符，实际部署时替换为 your_name。*

---

## 我们在做什么

专业术语是普通人理解世界的最大障碍。

"边际效用递减"、"负反馈回路"、"内稳态"——这些词在经济学、控制论、生物学里反复出现，说的其实是同一件事：**系统自稳定**。但每个学科各起各的名字，普通人根本不知道它们是一回事。

同物异名要做的事：**把不同领域里重复出现的结构找出来，翻译成日常用语，并用图数据库把这些知识连接起来**。

目前已有：
- 19,000+ 词条（977 已精加工 + 18,000+ 待精加工），覆盖 10 大学科
- 18,000+ 知识图谱关系（同义/抽象/简称/关联/对比）
- 18 组概念对比
- 26 个高薪岗位拆解
- 250+ Obsidian 协作词条，持续扩充中
- 双库架构：SQLite（精加工工作台）+ Neo4j（知识图谱展示）

---

## 为什么是图数据库

关系型数据库能存数据，但知识的本质不是表格，而是**网络**。

当你看到"负反馈"这个概念时，你想知道：它在控制论里是什么意思？在经济学里叫什么？在生物学里又是什么？它和"正反馈"是什么关系？和"内稳态"呢？

这种"从一个概念出发，沿着关系边探索整个知识网络"的体验，只有图数据库能原生支持。

| 能力 | 关系型数据库 | 图数据库 |
|------|-------------|---------|
| 多跳关联查询 | 层层 JOIN，性能急剧下降 | 原生图遍历，毫秒级响应 |
| 跨学科概念发现 | 需要复杂子查询 | `MATCH (a)-[:同构]->(b)` 一行搞定 |
| 知识路径推荐 | 几乎不可能 | 最短路径、共同邻居等原生算法 |
| 图谱可视化 | 需要大量应用层逻辑 | 直接导出节点+边 |

**我们要从 SQLite 迁移到图数据库（如 Neo4j），让知识真正"连"起来。**

## 路线图

### Phase 1：内容基础（已完成 ✅）

- 10 大学科词条框架，1069+ 词条入库
- 概念对比、岗位拆解模块
- Obsidian 协作流程（Markdown → Git → 数据库自动同步）
- Meilisearch 全文搜索
- 管理后台
- 关系型数据库中的基础图谱关系（语义关系表、学科关联表等）

### Phase 2：图数据库迁移（核心挑战 🔨）

**这是整个项目的技术转折点。**

目标：将知识数据从 SQLite 迁移到图数据库，释放知识的关联价值。

**技术选型（已确定）：**
- **图数据库**：Neo4j Community（社区版，Cypher 查询语言生态最成熟）
- **驱动**：neo4j-driver（Node.js 官方驱动）
- **迁移策略**：SQLite → Neo4j 数据迁移脚本 + 双写过渡期

**数据模型设计：**
- 节点类型：`词条`、`学科`、`职业`、`产业`、`岗位`、`别名`
- 边类型：`属于`、`关联`、`别名`、`对比`、`同构`
- 属性：每个节点携带翻译、解释、学科归属等

**已完成工作：**
- ✅ Neo4j 安装与配置（4.4.40 社区汉化版，Windows 服务）
- ✅ Neo4j 图数据库操作模块（db/图数据库.js）
- ✅ SQLite → Neo4j 数据迁移脚本（db/全量迁移到Neo4j.js）
- ✅ 图数据库业务服务层（services/图数据库服务.js）
- ✅ 图数据库 API 路由（routes/图数据库.js）
- ✅ 节点约束/索引（词条ID、学科名称等唯一约束）
- ✅ 全量数据迁移：19,703词条节点 + 10学科节点 + 18,000+关系
- ✅ 词库融合：19张词库表 → 10个核心学科（学科子领域映射表）
- ✅ 社区协作三层机制：学科映射管理 + 纠错建议 + Obsidian 协作

## 核心理念

> 不是词条数量，而是"发现不同领域在重复同一种结构"。

控制论的"负反馈"、经济学的"市场调节"、生物的"内稳态"——本质都是"系统自稳定"。
当普通人能一眼看穿这种结构相似性，就真正拥有了跨学科思考的能力。

**图数据库是实现这个愿景的关键。** 关系型数据库能存知识，但只有图数据库能让知识"连"起来，让用户在概念的网络中自由探索。
